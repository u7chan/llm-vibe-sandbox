# bomberman-clone

## 1. 目標
- シングルプレイヤーの最小限ボンバーマン。
- グリッド: 15x13 マス（NES風）。
- 勝利: 全敵（4体）撃破。
- 技術: HTML5 Canvas + JavaScript（ブラウザで動く）。

## 2. マップ仕様
```
15x13グリッド。各マス1x1ピクセル（スケール: 32px）。
- # (壊せない壁): 外周 + 内部ランダム配置（通れない）。
- ? (ソフトブロック): 内部ランダム20個（爆風で消滅、アイテム50%）。
- . (空きマス): 通行可能。
初期マップ例（JSON配列で定義）:
[
  ["#","#","#","#","#","#","#","#","#","#","#","#","#","#","#"],
  // ... 13行分（外周#、内部に#と?散在）
]
```

## 3. エンティティ
- **Player** (P): 位置(x,y), 爆弾数max=1, 火力=3, 速度=1マス/フレーム。
- **Enemy** x4: 位置(x,y), ランダム移動（壁/爆弾避け）。接触でPlayer死亡。
- **Bomb**: 位置(x,y), タイマー=180フレーム(3秒@60fps), 火力=Player火力。
- **Flame**: 爆発時一時生成（横縦直線火力マス分）。

## 4. 更新ルール (60fpsループ)
```
1. 入力処理:
   - 矢印キー: Player移動（壁/他Bomb貫通不可）。
   - Space: 空きマスにBomb設置（max数内）。

2. 移動:
   - Player/Enemy: グリッド移動（衝突判定）。

3. 爆弾処理:
   - タイマー-- ==0 → 爆発:
     * Flame生成（上下左右 fireマス、壁止まり）。
     * 範囲内: ソフトブロック消滅(50%アイテム), Enemy死亡, Player死亡。

4. 衝突:
   - Flame触れ: 即死亡（Player→ゲームオーバー、Enemy→消滅）。
   - Enemy接触Player: 死亡。

5. アイテム (Fire/Bomb-Up):
   - 取ったらPlayer火力/爆弾max +1。

6. 勝利/敗北:
   - Enemy==0 → WIN。
   - Player死亡 → GAME OVER (リトライ)。
```

## 5. 描画 (Canvas 480x416px)
- グリッド線なし。タイル: #=灰, ?=茶, .=黒。
- Player=白丸, Enemy=赤丸, Bomb=黒丸, Flame=黄線。
- UI: 残敵数, 火力/爆弾数表示。

## 6. 実装手順
1. Canvasセットアップ + ゲームループ(requestAnimationFrame)。
2. マップ2D配列ロード。
3. Entityクラス: Player, Enemy[], Bomb[], Flame[]。
4. 衝突関数: gridCollide(x,y)。
5. 爆発関数: explode(x,y,fire)。
6. キー入力: addEventListener。
7. 状態: playing, win, lose。

## 7. 必須最小機能のみ
- NO: 投げ/キック/リモコン/複数ライフ/サウンド。
- 敵AI: シンプル（ランダム4方向、壁避け）。
- デバッグ: console.log位置。
