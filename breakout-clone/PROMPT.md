# breakout-clone プロンプト

## 1. 目標
- シングルプレイヤーの最小限ブロック崩し。
- Canvas: 480x640 px（縦長）。
- 勝利: すべてのブロック破壊。
- 技術: HTML5 Canvas + JavaScript（ブラウザで動く）。

## 2. 画面・レイアウト
- Canvas サイズ: width=480, height=640。
- 上部にブロック群（複数行）、下部にプレイヤーパドル、中央/下部にボール。
- ブロック配置例:
  - 列: 10列、行: 5行（合計50ブロック）。
  - ブロックサイズ: width=44px, height=20px、左右に余白を含めて配置（列間margin=4px、行間margin=6px）。
- UI: 残ライフ (Lives)、スコア、残ブロック数を画面左上などに表示。

## 3. エンティティ
- Paddle（パドル）
  - 位置(x,y)、幅=80px、厚さ=12px、速度=8 px/frame（左右移動）。
  - 画面端で移動制限。
- Ball（ボール）
  - 位置(x,y)、半径=6px、速度ベクトル(vx, vy)。
  - 初期速度: vy = -4 px/frame, vx = 2 px/frame（方向ランダム化可）。
  - ボールは角度制御: パドルの当たり位置によって反射角を変える（中央は垂直、端は鋭角）。
  - 画面上/左右端で反射（反転）。
- Brick（ブロック）
  - 種類: 普通ブロック（1ヒットで破壊）。
  - 各ブロックは位置、サイズ、aliveフラグ、スコア値（例: 100点）。
  - Optional: 強化ブロック（2ヒット） は最小実装では不要。
- PowerUp（任意、最小実装ではオプション）
  - 取得するとパドル拡大、ボールスピード低下、ライフ+1、マルチボール等。
  - ブロック破壊時に一定確率で落下（例20%）。

## 4. ゲーム状態
- states: playing, paused, win, lose。
- Lives（残機）: 3（ボールを落とすと1減、0でGAME OVER）。
- Score（スコア）: ブロック破壊で加算。
- Win: 全てのブロックが破壊されたら勝利画面。

## 5. 更新ルール（60fps想定）
1. 入力処理:
   - 左/右矢印キーまたはA/Dでパドル移動（押下で速度方向）、マウスの左右移動で連動してパドル移動（画面端で停止）。
   - マウス移動でパドル追従。
   - スペースでゲーム開始/再開始（ボールのリリース）。
2. 移動:
   - パドル: 入力に応じてx更新（画面端制限）。
   - ボール: 位置 += velocity。
3. 衝突判定:
   - 壁: 左右反転、上辺反転。下辺を越すとボールロスト（ライフ減）。
   - パドル:
     * ボールがパドルに接触したらvy = -abs(vy)（上向きに）、
     * 当たり位置によってvxを調整（例: offset = (ball.x - paddle.center) / (paddle.width/2) → vx = offset * maxSpeedX）。
     * 反射後、速度を微調整して垂直バイアスを入れる。
   - ブロック:
     * 矩形衝突検出。衝突方向（上下/左右）に応じてvx/ vyを反転。ブロックは破壊（alive=false）し、スコア加算。
     * 破壊時にPowerUpドロップを判定（オプション）。
4. ライフ処理:
   - ボールが下に落ちたらライフ--。ライフ>0ならボールを初期位置にリセット、プレイヤーが再発射。ライフ==0でGAME OVER。
5. 勝利判定:
   - すべてのブロックが破壊 → WIN。

## 6. 描画（Canvas）
- 背景: 黒または濃いグレー。
- ブロック: 色は行ごとに変える（例: 赤・オレンジ・黄・緑・青）。
- パドル: 白または明るいグレーの長方形。
- ボール: 白丸。
- UI テキスト: スコア、残機、残ブロック数を固定位置に描画。
- アニメーション: requestAnimationFrameで描画ループ。

## 7. 実装手順（簡潔）
1. Canvasセットアップ + ゲームループ（requestAnimationFrame）。
2. ブロック配置を2D配列で生成（行数・列数・マージンで自動配置）。
3. クラス/オブジェクト: Paddle, Ball, Brick[], PowerUp[]（任意）。
4. 入力: addEventListener('keydown'/'keyup') と mousemove（オプション）。
5. 衝突関数: circleRectCollision(ball, rect) と反射処理。
6. ライフ/スコア管理、勝利/敗北判定と画面遷移（state管理）。
7. 描画関数: clearRect → draw bricks → draw paddle → draw ball → draw UI。
8. デバッグ: console.logで座標/衝突情報出力（必要時）。

## 8. 定数（推奨）
- FPS 想定: 60fps（requestAnimationFrameで可変）。
- Paddle:
  - WIDTH = 80
  - HEIGHT = 12
  - SPEED = 8
- Ball:
  - RADIUS = 6
  - INIT_VY = -4
  - INIT_VX = ±2
  - MAX_VX = 6
  - SPEED_INCREMENT_ON_HIT = 0.05（オプション）
- Bricks:
  - COLS = 10, ROWS = 5
  - BRICK_W = 44, BRICK_H = 20
  - MARGIN_X = 4, MARGIN_Y = 6
- Lives = 3
- PowerUp drop chance = 0.2（オプション）

## 9. 最小実装で外すべき機能（NO）
- NO: 複雑な物理演算、ボールの回転（スピン）、ステージ切り替え、複数ボール（最小は単一ボール）、音（任意）。
- Optional: アイテム/パワーアップや特殊ブロックは後回し。

## 10. 追加のヒント・挙動改善（任意）
- パドル中心付近でのヒットは垂直に跳ね返すことで難易度調整。
- ブロックの衝突方向判定は、ボールの前の位置とブロックの境界との重なり幅で決めると安定する。
- 衝突判定でボールがブロックに埋まらないように、反転後にボール位置を微調整して抜け出す。
